What are Skills?

Your own local scripts

Simple, transparent, controllable


📁 A folder on your machine

📄 SKILL.md - instructions for AI

⚙️ script.sh/.js/.py - your code


AI runs script → Gets output → Done

Why Skills?

✓ Key Benefits

  • 100% transparent - you read every line
  • Zero supply chain risk - no third-party code
  • No background daemons - runs & terminates
  • Full control - local AND external APIs
  • Wide support - works in most AI IDEs

What Can Skills Do?

Everything!


  • ✅ Local tasks (tests, builds, formatting)
  • ✅ External APIs (GitHub, Jira, Figma)
  • ✅ Database operations
  • ✅ File operations
  • ✅ Custom workflows
  • ✅ Browser automation

Use curl, fetch, libraries - whatever you need!

How Skills Work

  1. AI reads SKILL.md instructions
  2. AI decides to use the skill
  3. Runs your script: bash script.sh
  4. Reads output from stdout
  5. Script terminates - done!


Simple. Transparent. Secure.

Getting Started

# 1. Create folder
mkdir -p .skills/hello

# 2. Write SKILL.md
cat > .skills/hello/SKILL.md << 'EOF'
# Hello Skill
Simple test skill.
## Usage
bash .skills/hello/run.sh
EOF

# 3. Write script
cat > .skills/hello/run.sh << 'EOF'
#!/bin/bash
echo "Hello from Skills!"
echo "Time: $(date)"
EOF

chmod +x .skills/hello/run.sh

External API Example

# .skills/github-pr/create.sh
#!/bin/bash

TITLE=$1
BODY=$2

curl -X POST \
  -H "Authorization: token $GITHUB_TOKEN" \
  -H "Content-Type: application/json" \
  https://api.github.com/repos/owner/repo/pulls \
  -d "{
    \"title\": \"$TITLE\",
    \"body\": \"$BODY\",
    \"head\": \"feature-branch\",
    \"base\": \"main\"
  }"

Simple. Transparent. You control it.

Best Practices

✓ DO:

  • Keep skills simple and focused
  • Document in SKILL.md clearly
  • Handle errors gracefully
  • Use environment variables for secrets
  • Test skills manually first

✗ DON'T:

  • Hardcode credentials in scripts
  • Make skills too complex
